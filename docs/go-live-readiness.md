# Go-live readiness

O painel `Go-live` separa beta funcional de operacao publica.

Enquanto a prontidao beta mede o que ja pode ser fechado localmente, o go-live mede os criterios que precisam estar resolvidos para colocar o GitHub DevLog AI no ar como SaaS confiavel.

## URL no admin

- `/admin/go-live`

## Criterios acompanhados

- Dominio oficial com HTTPS.
- GitHub App oficial pronto para revisao.
- Mercado Pago em producao.
- Planos comerciais publicados.
- Email transacional real.
- Fila fora de `sync`.
- Documentacao publica suficiente.
- Suporte operacional.
- Roadmap publico/admin atualizado.
- Status publico e trilha de auditoria.

## Regra de decisao

O sistema so deve ser tratado como pronto para lancamento publico quando todos os criterios criticos estiverem como `ok`.

Antes disso, o produto pode seguir em beta, demo privada ou validacao com usuarios controlados.
