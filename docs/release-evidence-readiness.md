# Evidências de release

O painel `/admin/release-evidence` consolida os materiais que ajudam a demonstrar maturidade para lançamento, revisão externa e submissão ao GitHub Developer Program.

## O que ele verifica

- proposta pública do produto;
- documentação para usuários;
- referência de API;
- páginas legais e de segurança;
- contato e suporte;
- checklist do GitHub Developer Program;
- checklist de go-live;
- checklist técnico de produção;
- changelog público;
- sitemap e robots.

## Como usar

Antes de enviar o projeto para avaliação externa, abra o painel e confirme se todos os itens aparecem como `ok`.

Itens pendentes não significam necessariamente bug de código. Eles indicam ausência de uma peça de lançamento que pode ser exigida ou esperada por devs, avaliadores, parceiros ou usuários beta.
