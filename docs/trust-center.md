# Trust Center público

A página `/trust` centraliza a narrativa de confiança do GitHub DevLog AI.

## Objetivo

Dar a devs, times e revisores do GitHub Developer Program uma visão rápida sobre:

- isolamento por workspace;
- validação HMAC de webhooks GitHub;
- sanitização de payloads;
- auditoria operacional;
- suporte;
- status público;
- retenção e limites;
- governança de AI;
- pendências finais de go-live.

## Por que isso importa

Webhooks do GitHub podem carregar nomes de repositórios privados, emails, branches, commits, arquivos alterados e outros sinais sensíveis. Uma ferramenta de debug que não explica confiança parece só um RequestBin bonito. O Trust Center posiciona o DevLog AI como produto SaaS mais sério.
